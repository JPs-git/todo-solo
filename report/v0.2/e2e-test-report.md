## BUG 报告

### BUG #001：连续添加任务时第二个任务输入框被禁用

#### BUG 信息
- **严重程度**：高
- **类型**：功能缺陷
- **状态**：已确认

#### BUG 描述
连续添加两个新任务，第二个新任务无法修改任务标题等任何内容，也无法点击添加任务按钮。

#### 复现步骤
1. 点击添加新任务按钮
2. 输入任务名称（如"第一个任务"）
3. 点击添加任务按钮，成功添加
4. 再次点击添加新任务按钮
5. **问题出现**：所有输入显示禁用，无法点击添加任务按钮

#### 期望结果
可以连续添加任意个任务。

#### 实际结果
- 第二次点击"添加新任务"按钮后，表单中的输入框被禁用
- 输入框显示的是第一个任务的值（`value="第一个任务"`）
- 添加任务按钮处于禁用状态，无法点击

#### 测试验证
- **测试脚本**：`e2e/bug-verify.spec.ts`
- **测试结果**：失败（BUG 确认）
- **关键日志**：
  ```
  第二个任务输入框是否可用: false
  输入框元素属性: disabled, value="第一个任务"
  ```

#### 错误截图
测试失败截图已保存至：`test-results/bug-verify-BUG验证.../test-failed-1.png`

#### 可能原因分析（供开发人员参考）
根据测试结果分析，问题可能出在以下方面：
1. **表单状态未重置**：添加第一个任务后，表单状态（如 `editingTask`）未被正确重置
2. **输入框缓存问题**：第二次打开表单时，输入框可能保留了第一次的值和禁用状态
3. **React 状态管理问题**：`TaskForm` 组件可能在第一次提交后未正确卸载或重置

#### 建议排查方向
1. 检查 `App.tsx` 中 `editingTask` 状态的管理
2. 检查 `TaskForm` 组件在关闭后是否正确重置内部状态
3. 检查是否存在表单缓存或状态未清理的问题
