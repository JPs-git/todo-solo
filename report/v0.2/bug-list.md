### BUG-002: 全选按钮禁用状态逻辑问题

**状态**: ℹ️ 已确认非Bug

**严重程度**: 中

**描述**:  
测试脚本尝试点击已禁用的全选按钮，导致测试失败。

**分析结果**:  
经过代码审查和测试验证，发现这是测试脚本的逻辑问题，不是应用 Bug。

**应用行为**:  
- 当 `selectedCount === totalCount` 时，全选按钮被禁用（符合预期设计）
- 当 `selectedCount < totalCount` 时，全选按钮可用

**相关代码**:  
```typescript
// src/components/BatchToolbar.tsx
<button
  className="batch-toolbar__button batch-toolbar__button--secondary"
  onClick={onSelectAll}
  disabled={selectedCount === totalCount}  // 正确逻辑
  data-testid="select-all-button"
>
  全选
</button>
```

**修复措施**:  
- 更新测试脚本，移除对已禁用全选按钮的点击操作
- 改为验证按钮的禁用状态

### BUG-003: 任务编辑功能失效

**状态**: ✅ 已修复

**严重程度**: 高

**描述**:  
点击任务标题无法打开编辑表单，编辑功能完全失效。

**复现步骤**:
1. 创建一个任务
2. 点击任务标题
3. 编辑表单未显示

**期望结果**:  
点击任务标题后，编辑表单应该打开，可以修改任务内容

**实际结果**:  
点击任务标题无任何反应，编辑表单未显示

**修复方案**:  
- 为任务标题添加 `onClick={handleEdit}` 事件绑定
- 添加 `cursor: pointer` 样式提示可点击

### BUG-005: 空格键标记完成功能失效

**状态**: ✅ 已修复

**严重程度**: 中

**描述**:  
使用空格键快捷键无法标记任务为完成状态。

**复现步骤**:
1. 创建多个任务
2. 使用 Ctrl+A 全选任务
3. 按空格键

**期望结果**:  
所有选中的任务应该被标记为完成状态

**实际结果**:  
任务状态未改变，保持未完成状态

**问题原因**:  
选择复选框的 `onKeyPress` 事件拦截了空格键，调用 `e.preventDefault()` 和 `e.stopPropagation()` 阻止了事件冒泡到全局快捷键处理器。

**修复方案**:  
- 将选择复选框的事件处理从 `onKeyPress` 改为 `onKeyDown`
- 仅处理 Enter 键，不再拦截空格键
- 空格键事件现在可以正常冒泡到 window，触发全局标记完成功能

**修复文件**:  
- `src/components/TaskItem.tsx`

**验证结果**:  
✅ 第3次提测验证通过 - `e2e/bug-verify-round3.spec.ts`

**相关测试**:  
- `e2e/v02-integration.spec.ts` TC-V02-130
