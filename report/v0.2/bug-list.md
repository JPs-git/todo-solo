## Bug 详情

### BUG-001: 连续添加任务时表单状态未重置

**状态**: ✅ 已修复

**严重程度**: 高

**描述**:  
连续添加两个新任务，第二个新任务无法修改任务标题等任何内容，也无法点击添加任务按钮。

**复现步骤**:

1. 点击添加新任务按钮，输入任务名称，点击添加任务按钮，成功添加
2. 再次点击添加新任务按钮，所有输入显示禁用，无法点击添加任务按钮

**期望结果**:  
可以连续添加任意个任务

**实际结果**:  
第二次打开表单时，输入框被禁用，无法添加任务

**修复状态**:  
✅ 已修复 - 重新测试验证通过

**相关测试**:

- `e2e/bug-verify.spec.ts`
- `e2e/v02-integration.spec.ts` TC-V02-101

### BUG-002: 全选按钮禁用状态逻辑问题

**状态**: ℹ️ 已确认非 Bug

**严重程度**: 中

**描述**:  
测试脚本尝试点击已禁用的全选按钮，导致测试失败。

**分析结果**:  
经过代码审查和测试验证，发现这是测试脚本的逻辑问题，不是应用 Bug。

**应用行为**:

- 当 `selectedCount === totalCount` 时，全选按钮被禁用（符合预期设计）
- 当 `selectedCount < totalCount` 时，全选按钮可用

**相关代码**:

```typescript
// src/components/BatchToolbar.tsx
<button
  className="batch-toolbar__button batch-toolbar__button--secondary"
  onClick={onSelectAll}
  disabled={selectedCount === totalCount} // 正确逻辑
  data-testid="select-all-button"
>
  全选
</button>
```

**修复措施**:

- 更新测试脚本，移除对已禁用全选按钮的点击操作
- 改为验证按钮的禁用状态

**相关测试**:

- `e2e/v02-integration.spec.ts` TC-V02-121

### BUG-003: 任务编辑功能失效

**状态**: ❌ 待修复

**严重程度**: 高

**描述**:  
点击任务标题无法打开编辑表单，编辑功能完全失效。

**复现步骤**:

1. 创建一个任务
2. 点击任务标题

**实际结果**:  
点击任务标题无任何反应，编辑表单未显示
