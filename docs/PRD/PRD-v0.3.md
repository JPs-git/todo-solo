# Todo-List v0.3 版本产品需求文档

## 1. 产品概述

### 1.1 产品简介
Todo-List 是一款简洁高效的任务管理应用，帮助用户记录、管理和跟踪日常任务。v0.3版本将基于用户反馈进行优化，提升用户体验和系统性能。

### 1.2 版本信息
| 项目 | 内容 |
|------|------|
| 版本号 | v0.3 |
| 规划日期 | 2026-03-09 |
| 目标发布日期 | 2026-03-20 |
| 需求来源 | v0.2用户反馈 |

### 1.3 设计理念
- **用户体验优先**：基于用户反馈优化操作流程和界面设计
- **功能完整性**：补充核心功能，如废纸篓、操作统一等
- **技术优化**：使用现代化前端技术提升性能和可维护性
- **扩展性**：为未来功能预留扩展空间

## 2. 核心功能

### 2.1 功能总览

| 功能模块 | 功能名称 | 优先级 | 功能描述 |
|---------|---------|---------|---------|
| **用户体验优化** | 批量操作UI优化 | 高 | 将多选框与完成框分离，独立批量操作功能 |
| | 模态框统一 | 高 | 使用Ant Design封装模态框组件，统一交互体验 |
| | 样式系统优化 | 高 | 使用Tailwind CSS进行样式管理，减少CSS体积 |
| **核心功能增强** | 废纸篓功能 | 高 | 删除任务先存放到废纸篓，支持恢复和永久删除 |
| | 撤销操作 | 中 | 支持撤销删除、撤销完成等操作 |
| | 数据导入导出 | 中 | 支持JSON/CSV格式数据导入导出 |

### 2.2 详细功能描述

#### 2.2.1 批量操作UI优化
- **功能描述**：将批量操作功能从任务项中独立出来，避免与完成框混淆
- **操作流程**：
  1. 顶部工具栏添加「批量操作」按钮
  2. 点击后进入批量选择模式
  3. 任务项显示独立的选择框
  4. 支持全选/取消全选
  5. 底部显示批量操作工具栏（删除、标记完成/未完成）
- **UI设计**：
  - 顶部工具栏添加批量操作入口
  - 任务项左侧显示独立选择框（与完成框区分）
  - 底部批量操作栏固定显示

#### 2.2.2 模态框统一
- **功能描述**：使用Ant Design封装模态框组件，统一所有模态框的样式和交互
- **覆盖场景**：
  - 添加新任务
  - 编辑任务
  - 删除确认
  - 批量操作确认
- **技术实现**：
  - 引入Ant Design Modal组件
  - 封装统一的模态框组件
  - 确保所有模态框外观一致

#### 2.2.3 样式系统优化
- **功能描述**：使用Tailwind CSS替代传统CSS，减少样式文件体积
- **优化目标**：
  - 减少CSS文件大小（目标：< 10KB）
  - 提高样式开发效率
  - 确保响应式设计一致性
- **技术实现**：
  - 引入Tailwind CSS
  - 重构现有样式
  - 配置自定义主题

#### 2.2.4 废纸篓功能
- **功能描述**：删除的任务自动进入废纸篓，支持恢复和永久删除
- **操作流程**：
  1. 删除任务时，任务移至废纸篓
  2. 侧边栏添加「废纸篓」入口
  3. 废纸篓中显示所有已删除任务
  4. 支持单个/批量恢复任务
  5. 支持单个/批量永久删除
  6. 支持清空废纸篓
- **数据存储**：
  - 废纸篓数据单独存储（localStorage）
  - 保留删除时间和原任务信息

#### 2.2.5 撤销操作
- **功能描述**：支持撤销最近的操作，如删除、标记完成等
- **支持操作**：
  - 撤销删除（恢复任务）
  - 撤销标记完成（恢复为未完成）
  - 撤销标记未完成（恢复为完成）
- **操作方式**：
  - 快捷键：Ctrl+Z
  - 顶部工具栏：撤销按钮
  - 操作后显示撤销提示

#### 2.2.6 数据导入导出
- **功能描述**：支持任务数据的导入和导出，方便备份和迁移
- **支持格式**：
  - 导出：JSON、CSV
  - 导入：JSON、CSV
- **操作流程**：
  1. 顶部菜单添加「导出数据」选项
  2. 选择导出格式
  3. 下载数据文件
  4. 支持拖拽文件导入
  5. 导入前预览和确认

## 4. 数据模型

### 4.1 任务数据结构

```typescript
interface Task {
  id: string;
  title: string;
  description: string;
  completed: boolean;
  priority: 'low' | 'medium' | 'high';
  tags: string[];
  dueDate: string | null;
  createdAt: string;
  updatedAt: string;
}
```

### 4.2 废纸篓数据结构

```typescript
interface TrashTask {
  task: Task;
  deletedAt: string;
}
```

### 4.3 应用状态结构

```typescript
interface AppState {
  tasks: Task[];
  trash: TrashTask[];
  filter: 'all' | 'completed' | 'active';
  sortBy: 'createdAt' | 'completed' | 'priority' | 'dueDate' | 'order';
  searchTerm: string;
  selectedTag: string | null;
  selectedTaskIds: string[];
  isBatchMode: boolean;
  undoStack: Action[]; // 撤销操作栈
}
```

## 8. 风险评估

### 8.1 技术风险
- **Tailwind CSS迁移**：可能需要重构现有样式，影响开发进度
- **Ant Design集成**：与现有组件的兼容性问题
- **数据结构变更**：可能影响现有数据的迁移

### 8.2 解决方案
- **分阶段迁移**：先保持现有样式，逐步迁移到Tailwind
- **组件封装**：统一封装Ant Design组件，确保兼容性
- **数据迁移脚本**：编写数据迁移工具，确保平滑升级
