## 修复总结

### 问题分析

初始测试执行时，25 个测试因元素选择器不匹配而失败。主要问题包括：

1. **表单字段选择器**: 使用了 `input[name="title"]` 而实际使用 `id="title"`
2. **按钮选择器**: 使用了 `button[type="submit"]` 而实际使用 `.btn-primary` 类
3. **模态框选择器**: 使用了 `.modal:has-text("确认删除") .btn-primary` 而实际使用 `.modal__btn--primary`

### 修复措施

1. **更新表单字段选择器**:

   - `input[name="title"]` → `#title`
   - `textarea[name="description"]` → `#description`
   - `input[name="dueDate"]` → `#dueDate`
   - `input[name="tags"]` → `#tags`

2. **更新按钮选择器**:

   - `button[type="submit"]` → `.btn-primary:has-text("添加任务")` / `.btn-primary:has-text("更新任务")`
   - `.modal__confirm-btn` → `.modal__btn--primary`

3. **使用更精确的选择器**:
   - 使用 `:has-text()` 伪类来定位包含特定文本的元素
   - 使用 CSS 类选择器替代属性选择器
